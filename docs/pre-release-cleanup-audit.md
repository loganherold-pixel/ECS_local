# ECS Pre-Release Cleanup Audit

Date: 2026-05-02

## Scope And Method

This audit was intentionally conservative. It did not treat "not imported by an active route" as automatic dead code because ECS has protected future systems and release gates that are intentionally present before full UI wiring.

Checks performed:

- Built a relative import graph from active Expo Router app files, excluding `app/dev`.
- Built a second graph from package script entrypoints, including release gates and smoke tests.
- Compared `components/`, `lib/`, `src/`, `stores/`, `context/`, and `config/` files against those graphs.
- Scanned root generated artifacts, duplicate basename clusters, dev-only routes, legacy feature keywords, and stale feature flag patterns.
- Ran TypeScript and lint as validation gates after the low-risk cleanup.

Snapshot:

- Source/script files scanned: 1469
- Active app route roots: 51
- Files reachable from active app routes: 942
- Package script roots: 191
- Files reachable from package scripts: 409
- Candidate source files outside active app route graph: 236
- Candidate source files outside active app and package-script graphs: 235
- Root generated artifact candidates: 80
