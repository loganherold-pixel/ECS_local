# Safe Cleanup Utility

`scripts/safe-cleanup.mjs` is a reversible cleanup helper for the ECS app folder. It defaults to read-only dry-run mode and only handles the approved low-risk candidates from the cleanup plan.

## What It Can Clean

The script can quarantine these known generated/local artifacts:

- `.npm-cache/`
- `.expo-export-check/`
- `.expo-runtime/`
- `.gradle-local/`
- `.gradle-local-build3/`
- root temporary screenshots matching `.codex-*.png`, `tmp_*.png`, or `tmp-*.png`

These paths are untracked/ignored local outputs from the audit. They can be regenerated by normal package, Expo, smoke-test, or Android build workflows.

## What It Will Never Clean

The utility fails closed. It will not quarantine unknown large files or paths outside its explicit allowlist. It also has a denylist for:

- `.env*`, secrets, credentials, keys, signing files
- databases such as `*.db`, `*.sqlite`, `*.sqlite3`, `*.realm`
- uploads, storage, media, user-generated files
- backups, exports, Docker volumes
- Supabase migrations and functions
- source directories such as `app/`, `components/`, `lib/`, `src/`, `stores/`, `scripts/`
- `assets/`, `public/`, docs, tests, fixtures
- lockfiles, app config, build/deployment config
- `.git/`
- medium/high-risk cleanup candidates such as `node_modules/`, `dist/`, `artifacts/`, `android/app/build/`, `.gradle-user-home/`, and release/field-test Gradle homes

## Working Tree Guard

The script refuses to run when `git status --short` is dirty unless `--allow-dirty` is passed. This is intentional: cleanup should be easy to review and should not hide unrelated worktree changes.

## Dry Run

Dry-run is the default mode:

```powershell
node scripts/safe-cleanup.mjs --allow-dirty
```

Equivalent explicit form:

```powershell
node scripts/safe-cleanup.mjs --dry-run --allow-dirty
```

The report includes branch baseline detection, candidate decisions, size estimates, git ignore/tracking evidence, and detected validation commands.

## Report Only

Use `--report` when you want a markdown report without moving files:

```powershell
node scripts/safe-cleanup.mjs --report --allow-dirty
```

Generated report format:

```markdown
# ECS Safe Cleanup Report

- Mode
- App path
- Baseline
- Package manager
- Reclaimable approved candidate size

## Candidates

| Path | Size | Decision | Git status | Evidence |

## Validation Commands

| Type | Command |

## Notes
```

## Quarantine

Quarantine moves approved candidates to a timestamped folder beside the app folder by default:

```powershell
node scripts/safe-cleanup.mjs --quarantine --allow-dirty
```

Default quarantine root:

```text
../ECS_local-cleanup-quarantine/
```

Each quarantine run writes:

- `manifest.json`
- `report.md`
- `safe-cleanup.log`
- `files/` containing the moved paths

You can choose a different quarantine root, but it must be outside the app folder and include `cleanup-quarantine` in its path:

```powershell
node scripts/safe-cleanup.mjs --quarantine --allow-dirty --quarantine-dir ..\ECS_local-cleanup-quarantine
```

## Restore

Restore uses the latest manifest under the quarantine root unless `--quarantine-dir` points directly at a specific run folder:

```powershell
node scripts/safe-cleanup.mjs --restore --allow-dirty
```

Restore refuses to overwrite existing paths.

## Permanent Delete

Only delete quarantine after manual approval and successful validation:

```powershell
node scripts/safe-cleanup.mjs --delete-quarantine --confirm-delete --allow-dirty
```

This deletes the quarantine folder only. It never deletes cleanup candidates directly from the app folder.

## Validation Commands

The repo uses npm because `package-lock.json` exists. The script detects:

```powershell
npm run lint
npm run build
npm run smoke
```

There is no generic `npm test` script in `package.json`; targeted `test:*` scripts exist and should be selected based on the feature area being validated. `npm run start` exists but is long-lived, so it is reported rather than selected for automatic validation.
